# Emoji Operations

## 목적

- 채팅 이모지 검색과 반응 기능이 외부 API를 어떻게 쓰는지 짧게 정리한다.

## 현재 동작

- 서버는 EmojiHub 공개 API를 조회한다.
- 전체 목록은 서버 메모리에 약 6시간 캐시한다.
- 검색어가 없으면 기본적으로 `smileys and people` 범주를 먼저 보여준다.

## 운영 메모

- 이모지 검색이 느리면 첫 요청인지, 캐시가 이미 채워졌는지 먼저 확인한다.
- 외부 API 장애 시 `/api/emojis`가 실패할 수 있으므로, 사용자는 입력/전송 자체는 계속 가능해야 한다.
- 검색 결과가 이상하면 `name`, `group`, `category` 기준 부분 검색이라는 점을 확인한다.
